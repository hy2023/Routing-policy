# Mihomo 私人yaml稳定ip分流配置方案

两个配置文件：`fallback-all.yaml`与 `fallback-smart.yaml`

设计核心在于**解决节点频繁自动切换（跳 IP）导致被各类应用（尤其是 AI 平台、流媒体、社交软件）判定为异地登录或账号异常而触发风控的问题**。

---

## 💡 为什么这两个配置能解决“频繁跳 IP”？

传统的 `url-test`（自动测速）模式会因为网络波动，在短时间内高频切换到延迟最低的节点，导致你的公网 IP 不断变化，极易触发 ChatGPT、Google、Netflix 等平台的安全风控。

本方案通过以下两种不同的逻辑完美解决了这一痛点：

### 1. `fallback-all.yaml`
* **核心逻辑**：采用 `fallback`（故障转移）作为区域组的默认核心策略。手选组作为固定ip策略,url-test(自动测速)作为备选策略。
* **工作机制**：只要当前手动选定的可用节点**没有死线（断网）**，即使它的延迟从 50ms 波动到 150ms，配置也**绝不会**主动切换节点。只有当该节点彻底断开时，才会通过故转顺延到下一个备用节点。当手选节点恢复可用再自动恢复回来。

### 2. `fallback-smart.yaml`(需开启smart内核)
* **核心逻辑**：同上,只是把备选url-test(自动测速)换成了smart方案
* **工作机制**：利用算法（如 LightGBM 权重调整）智能选择最优备用节点
